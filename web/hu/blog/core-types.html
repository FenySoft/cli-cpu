Öt mag, egy chip

A CFPU core család: Nano, Actor, Rich, Matrix, Seal
2026. április 18. · Hocza József Szabolcs

Kettős specializáció

A CFPU nem egyetlen mag milliósszorosítása. Különböző feladatok különböző processzor-geometriát igényelnek — ezért a CFPU core család két irányban ágazik el egyetlen közös alapból, a Nano Core-ból.

Az első ág az aktor-feldolgozás irányába nő: Nano → Actor → Rich. Minden lépés nagyobb SRAM-ot, több regisztert és gazdagabb ISA-t jelent — az Actor általános célú CIL végrehajtásra, a Rich lebegőpontos számításra és GC-vel felügyelt objektumkezelésre optimalizált.

A második ág az ML/SNN compute irányába ágazik: Nano → Matrix. A Matrix Core nem az Actor vagy a Rich méretezett változata — hanem a Nano egyenes leszármazottja, MAC+SFU egységekkel bővítve. Kis, tömeges, streamelhető. 

Nano Core (alap)
  ├── Actor Core  (általános aktor, 64 KB SRAM)
  │     └── Rich Core   (FP + GC + Obj, 256 KB SRAM)
  └── Matrix Core (ML/SNN compute, 8 KB SRAM + MAC+SFU)

Seal Core (biztonsági mag, különálló fejlődési ág)

Ez a kettős elágazás nem véletlen: a Rich Core drága (256 KB SRAM, GC, FPU), míg a Matrix Core célja, hogy minél több férjen egy chipre. A Nano-tól való közvetlen leszármazás tartja alacsonyan a területigényt.

Az öt core típus

Nano Core — spike / érzékelő

A legkisebb és legtömegesebb mag. Csak egész aritmetika, 4 KB SRAM, minimális ISA, mailbox-alapú üzenetkezelés. Érzékelőkhöz, edge-feldolgozáshoz és SNN spike-propagációhoz tervezve — ahol nem lebegőpontos precízió kell, hanem sebesség és energiahatékonyság.

Actor Core — általános aktor

A CFPU fő munkása. Teljes CIL ISA, GC-vel és objektumkezeléssel, 64 KB SRAM, mailbox-alapú üzenetkezelés. Neuron OS aktorok, szerver logika, általános célú párhuzamos feldolgozás. Ez a legtöbb CFPU-konfigurációban a domináns magtípus.

Rich Core — FP aktor

Az Actor Core kiterjesztése lebegőpontos egységgel (FPU). 256 KB SRAM. Numerikus számítás, tudományos kódok, összetett üzleti logika. Az összehasonlítási alap minden külső CPU-val szemben.

Matrix Core — ML/SNN compute

MAC+SFU (Multiply-Accumulate + Special Function Unit) kiegészítéssel bíró Nano-leszármazott. 8 KB SRAM, streaming adatfolyam. Mátrix szorzásra, konvolúcióra és Spiking Neural Network propagációra optimalizált. Tömeges párhuzamosság: 7nm-en ~19 000 mag fér egy 800 mm²-es chipre (routerrel és infrastruktúra overhead-del együtt).

Seal Core — hitelesítés

Biztonsági mag, a chip közepén, az L3 crossbar-ral egybeépítve. Minden kereszt-régió forgalom természetesen áthalad rajta. Kódhitelesítés, kulcskezelés, izolációs határ-ellenőrzés. Nem számítási mag — biztonsági kapuőr.

Matrix Core: a Nano steroidokon

Az első kérdés, ami felmerül: miért nem a Rich Core-ból fejlesztettük ki a Matrix Core-t? A Rich-nek van FPU-ja, nagy SRAM-ja — nem ez kell ML-hez?

A válasz a területhatékonyság. Egy Rich Core 256 KB SRAM-mal és GC-vel ~107 000 gate. Ugyanezen a területen háromszor annyi Matrix Core fér el — és a ML/SNN workload nem egyetlen nagy mag gyorsaságától, hanem a párhuzamos MAC-műveletek számától függ.

A Matrix Core 8 KB SRAM-ja nem hátránya — a streaming feldolgozás édespontja. Egy konvolúciós réteg vagy SNN propagáció tipikusan kis lokális bufferben fut, majd tovább adja az eredményt. A 8 KB pont elegendő egy tile feldolgozásához. Nagyobb SRAM = pazarlás.

A Matrix Core nem a Rich Core FPU nélküli változata, hanem a Nano Core MAC+SFU-val kiegészítve — és ez a különbség 3× több magot jelent ugyanazon a chipen.

A MAC+SFU egység (szorzás-összeadás + sigmoid/tanh/ReLU hardver) közvetlenül a CIL pipeline-ba illeszkedik. Az integer Nano ISA megtartása biztosítja, hogy a vezérlési logika (loopok, feltételek, üzenetküldés) ugyanolyan hatékony marad.

A Seal Core egyetlen példányban van jelen chipeánként (L3 crossbar közepén), ezért nem szerepel a magszám-táblában. A magszámok tartalmazzák az L0 router területet és az L1–L3 infrastruktúra overhead-et (~20%).

CFPU-ML: az ML/SNN chip

CFPU-ML konfiguráció (7nm, 800 mm², becsült):

~19 500 Matrix Core — MAC+SFU streaming tömb (8 KB SRAM, Turbo router)
~1 000 Actor Core — supervisor, scheduler, koordináció (64 KB SRAM)
1 Seal Core — modell hitelesítés, izoláció

Peak throughput (csúcsteljesítmény): ~312 INT8 TOPS* — elméleti maximum, minden MAC minden ciklusban dolgozik
Sustained throughput (tartós teljesítmény): ~49–176 INT8 TOPS* — reális tartomány az optimalizálási stratégiától függően

Fő előny: Minden Matrix Core önálló aktor — nincs shared memory, nincs driver stack, Neuron OS-ről közvetlenül programozható CIL üzenetekkel.

* Peak = core szám × MAC egységek × órajel. A sustained tartomány az SRAM mérettől és az adatbetáplálási stratégiától függ (weight reuse, systolic láncolás, post-MAC feldolgozás). A MAC tömb ~6×-osan gyorsabb, mint amennyit a hálózat táplálni tud — a sustained throughput hálózat-korlátozott, nem compute-korlátozott. RTL szimuláció vagy szilícium mérés még nem áll rendelkezésre.

A CFPU-ML nem GPU-versenytárs — hanem más paradigma. Ahol a GPU egy hatalmas, monolitikus shader-tömb, a CFPU-ML ~20 000 független ML-aktor, amelyek üzenetekkel kommunikálnak. Részleges hibák esetén az egészséges magok tovább futnak; nincs globális memória, nincs pointer-manipuláció, nincs CUDA/ROCm driver stack.

Miért CFPU-ML?

A CFPU-ML értéke nem a nyers TOPS-ban rejlik, hanem olyan architekturális tulajdonságok kombinációjában, amelyet egyetlen meglévő chip sem kínál együtt: programozható neuron modell (tetszőleges CIL), event-driven működés (0W idle), shared memory nélküli izolált core-ok, nyílt forráskódú hardver (CERN-OHL-S), és natív .NET CIL végrehajtás.

A CFPU-ML-Max variáns — Systolic Wide routerrel, post-MAC pipeline-nal és 1 GHz órajellel — 90–95% sustained MAC kihasználtságot ér el, és azonos die méret mellett 2–3,7× jobb sustained TOPS-t ad, mint az NVIDIA RTX 4060/4090.
